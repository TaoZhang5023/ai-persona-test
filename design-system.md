# AI 使用人设测试 —— 前端设计规范

> 统一的视觉语言，任何新页面/组件都按这份规范来做，保证风格一致。

## 一、设计定位

**风格关键词：Retro-Zine × Terminal Playful**（复古小志 × 终端趣味）

- **温度感**：奶黄底 + 墨黑字，避免纯白+纯黑的冷硬感，让"测试"读起来像一本小杂志
- **仪式感**：维度字母、人设代号、进度数字全部用等宽字体处理，像 IDE / 终端界面，强化"体系化测试"的可信度
- **活力感**：橙红 + 紫 + 黄的高饱和点缀色，不会被误认成严肃心理测试
- **触感**：卡片使用 2px 硬边 + 硬阴影的轻拟物风格（neo-brutalism lite），点按时位移塌缩，反馈强
- **手机优先**：单列布局，所有可点击元素 ≥ 48px，顶部粘性进度条

**调性约束**：娱乐向、自嘲、略带赛博口吻。避免：可爱卡通、商务沉稳、极简性冷淡。

## 二、色彩系统

### 基础色

| 变量 | 色值 | 用途 |
|------|------|------|
| `--bg` | `#FAF6EE` | 页面主背景（奶黄） |
| `--bg-2` | `#F2ECDD` | 外壳 / 进度条底色（深一档奶色） |
| `--ink` | `#161513` | 主文字色（墨黑，不是纯黑） |
| `--ink-2` | `#3A362E` | 次级文字（说明、描述） |
| `--muted` | `#7A7366` | 弱化文字（标签、caption） |
| `--line` | `#1A1918` | 描边 / 硬阴影色 |
| `--card` | `#FFFFFF` | 卡片背景 |

### 强调色

| 变量 | 色值 | 用途 |
|------|------|------|
| `--primary` | `#FF5A1F` | 主 CTA、进度条填充、强调数字、关键字母 |
| `--accent` | `#6B5AED` | 维度标签、数据点缀、次级字母 |
| `--accent-2` | `#0FB5A6` | 备用青绿（极少使用） |
| `--yellow` | `#FFD23F` | 高亮划线、印章、彩蛋暗示 |
| `--pink` | `#FFB3B3` | 备用柔色 |

### 使用原则

1. 一屏内强调色不超过 2 种（主色 + 黄色高亮 或 主色 + 紫色数据）
2. 大面积色块永远是 `--bg`，强调色只用在点、线、字上
3. 文字永远优先用 `--ink`，不要用纯黑 `#000`
4. 彩蛋人设场景允许使用 `--primary` → `--accent` 的渐变突出"超越体系"的感觉

## 三、字体系统

### 字体家族

```css
--font-sans: "Noto Sans SC", "PingFang SC", -apple-system, system-ui, sans-serif;
--font-mono: "JetBrains Mono", ui-monospace, Menlo, monospace;
```

Google Fonts 引入：

```html
<link href="https://fonts.googleapis.com/css2?family=JetBrains+Mono:wght@500;700;800&family=Noto+Sans+SC:wght@400;500;700;900&display=swap" rel="stylesheet" />
```

### 何时用等宽 Mono

| 用 Mono | 不用 Mono |
|---------|----------|
| 四字母人设代号（`PCIS`） | 正文描述 |
| 维度字母（`P / W`） | 标题 |
| 题号（`01 / 28`） | 按钮文字 |
| 进度百分比（`67%`） | 人设名 |
| 小标签（`28 QUESTIONS`、`DIM · 调教流派`） | tagline |
| 品牌标识（`AI.PERSONA / v1.0`） | |

**规则**：凡是"数据化、编号化、代号化"的文本 → Mono。凡是"给人读的句子" → Noto Sans SC。

### 字号与字重阶梯

| 用途 | 字号 | 字重 | 备注 |
|------|------|------|------|
| 封面大标题 | 40–44px | 900 | 字间距 -0.01em |
| 人设代号 | 80–96px | 800 | Mono，字间距 0.04em |
| 人设名 | 28–32px | 900 | |
| 题目 | 20–22px | 700 | 行高 1.45 |
| 正文描述 | 15–16px | 400 | 行高 1.6–1.7 |
| 按钮 | 17px | 700 | |
| 次级说明 | 13–14px | 400 | 色 `--ink-2` |
| 小标签 | 11–12px | 700 | Mono，letter-spacing .1–.2em，全大写 |
| Caption | 11px | 500 | 色 `--muted` |

## 四、间距与圆角

### 间距

使用 4 的倍数：`4 / 8 / 10 / 12 / 14 / 16 / 18 / 20 / 24 / 32`

- 组件内部 padding：16–20px
- 卡片之间的间距：10–14px
- 段落之间：16–20px
- 屏幕左右边距：20px（固定）

### 圆角

| 变量 | 值 | 用途 |
|------|----|----|
| `--radius` | `18px` | 大卡片、主按钮 |
| `--radius-sm` | `12px` | 小卡片、选项、dim 块 |
| — | `999px` | 标签 pill、进度条 |
| — | `4px` | 印章、小高亮块 |

**不使用**：直角（太冷硬）、极大圆角（胶囊除外，太 Q）。

## 五、描边与阴影

这是本设计系统的**灵魂特征**，务必统一执行。

### 硬边 + 硬阴影

```css
--shadow:    4px 4px 0 0 var(--line);   /* 大卡片、主按钮 */
--shadow-sm: 3px 3px 0 0 var(--line);   /* 小卡片、选项 */

border: 2px solid var(--line);           /* 所有需要"有重量"的卡片 */
border: 1.5px solid var(--line);         /* 标签、小元素 */
```

### 按压反馈（所有可点击卡片都要加）

```css
.clickable {
  transition: transform .12s ease, box-shadow .12s ease;
}
.clickable:active {
  transform: translate(3px, 3px);
  box-shadow: 1px 1px 0 0 var(--line);
}
```

原理：按下时卡片整体位移 3px 到原阴影位置，阴影缩小，模拟"把立体的东西按平"。

### 禁止使用

- 模糊软阴影（`box-shadow: 0 4px 12px rgba(0,0,0,.1)`）—— 与整体风格冲突
- 无描边的浮动卡片 —— 会"飘"，失去重量感
- 描边色用 `#000` —— 要用 `--line`

## 六、核心组件规范

### 1. 按钮 `.btn`

```
高度: ≥ 54px
内边距: 16px 24px
边框: 2px solid var(--line)
圆角: 18px
阴影: 4px 4px 0 0 var(--line)
字号: 17px / 700
```

**三种变体**：

- `.btn.primary` — 橙红底白字，页面主 CTA（"开始测试"、"分享"）
- `.btn.outline` — 奶黄底墨黑字，次级操作（"重新测试"）
- `.btn` 默认 — 墨黑底白字，用在辅助场景
- `.btn.ghost` — 无边无底，仅文字，纯文字链接用

### 2. 卡片 `.card`

```
背景: var(--card) 或 var(--bg)
边框: 2px solid var(--line)
圆角: 18px (大) / 12px (小)
阴影: 硬阴影
内边距: 18–20px
```

### 3. 标签 `.tag`

Pill 形状，Mono 字体，全大写，letter-spacing 0.08–0.1em。

```
高度: 22–26px
内边距: 5px 10px
边框: 1.5px solid var(--line)
圆角: 999px
字号: 11px / 700
```

变体：默认（米黄）/ `.solid`（墨黑底）/ `.primary`（橙红底）/ `.accent`（紫底）

### 4. 选项卡片 `.option`

```
布局: 圆形字母徽章 + 文本，flex 对齐 flex-start
字母徽章: 28×28 圆，Mono 字体，1.5px 描边
选中态: 整卡变墨黑底白字，字母徽章变橙红
```

### 5. 进度条 `.progress`

```
高度: 8px
底色: var(--bg-2)
边框: 1.5px solid var(--line)
圆角: 999px
填充色: var(--primary)
过渡: width .4s cubic-bezier(.4,0,.2,1)
```

### 6. 维度条 `.dim-bar`

比进度条粗一点（14px），内部显示百分比文字（Mono 10px 800）。

## 七、版式模板

### 顶部信息栏

左侧：品牌标识（Mono 12px + 橙色方块前缀）
右侧：状态标签（题目总数 / 结果类型）

```
▪ AI.PERSONA / v1.0        [ 28 QUESTIONS ]
```

### 封面页结构

1. 顶部信息栏
2. 大标题（含黄色高亮下划线 + 橙色星标装饰）
3. 副标题（2 行，色 `--ink-2`）
4. Hero Card（展示人设代号示例 + 中文人设名，字母轻微浮动动画，下方显示对应中文名如「AI 驯兽师」）
5. 四维度 2×2 网格
6. Meta 标签行（时长 / 彩蛋 / 属性）
7. 底部主 CTA 按钮 + 免责声明

### 答题页结构

1. 粘性顶部：返回箭头 · 题号 · 维度标签 + 进度条
2. 维度 caption（`DIM · 调教流派`）
3. 题目（22px 700，行高 1.45）
4. 4 个选项卡片（点击后有选中态 → 延迟 260ms 自动下一题）

### 结果页结构

1. 顶部信息栏（彩蛋时显示 🥚 EGG 标签）
2. Hero 区：
   - Caption `你的人设代号`
   - 巨型四字母代号（每字母不同颜色，彩蛋则为 emoji）
   - 人设名（30px 900）
   - Tagline（14px，最多 2 行，色 `--ink-2`）
3. 人设画像卡（含印章装饰）
4. 四维度画像卡（4 条维度条 + 百分比）
5. 代表语录卡
6. 底部双按钮：分享（主 CTA）/ 重测

## 八、动效规范

| 场景 | 效果 | 时长 / 缓动 |
|------|------|------------|
| 页面切换 | 淡入 + 向上 8px 位移 | 350ms ease |
| 按钮点击 | 位移到阴影位置 | 120ms ease |
| 选项选中 → 下一题 | 选中态高亮 → 延迟跳转 | 260ms |
| 进度条增长 | width 过渡 | 400ms cubic-bezier(.4,0,.2,1) |
| 维度条填充 | width 从 0 增长 | 800ms cubic-bezier(.4,0,.2,1) |
| 首页代号字母浮动 | 上下 6px 循环 | 3s ease-in-out infinite，每字母延迟 .2s |

**禁用**：弹跳 bounce、旋转 360、粒子效果。保持克制。

## 九、图标与装饰元素

- **星标** `✦`：封面大标题末尾，橙色 + 旋转 -8deg，作为视觉落点
- **方块前缀** `▪`：品牌标识前，橙色
- **实心圆** `●`：卡片小标题前，橙色 8×8px 带描边
- **引号**：描述中保留中文 `「」` 或 `""`，不做特殊替换
- **装饰性渐变**：仅用于 Hero Card 的角落径向渐变背景，10–15% 透明度
- **印章 `.stamp`**：虚线边框 + 橙色文字 + 旋转 -3deg，用于"CERTIFIED"等强调场景

## 十、手机端适配要点

```css
padding: calc(20px + env(safe-area-inset-top)) 20px calc(24px + env(safe-area-inset-bottom));
```

- 所有滚动容器顶部留安全区
- 最大宽度 420px，超出时居中显示（桌面预览时模拟手机壳）
- 禁用缩放：`<meta name="viewport" content="width=device-width, initial-scale=1, maximum-scale=1, user-scalable=no">`
- 关闭点击高亮：`-webkit-tap-highlight-color: transparent`
- 文字选中色：`::selection { background: var(--yellow); color: var(--ink); }`
- 横屏兼容：使用 `min-height: 100dvh` 而不是 `100vh`

## 十一、文案风格

- **自嘲 > 夸赞**：描述用户时多用反讽，少用正面形容词
- **具体 > 抽象**：说"蹲马桶 40 分钟跟 AI 争论"，别说"高频使用"
- **场景化**：题目永远给出一个具体情境（"周一早上老板突然让你…"）
- **克制 emoji**：仅在彩蛋人设（🧠💕⏳🔓）和 meta 行（⏱🎁）使用
- **标点**：破折号用 `——`，省略号用 `…`，引号用 `""` 或 `「」`

## 附：改版时的决策原则

遇到拿不准的样式选择，按以下优先级决策：

1. **与现有 preview.html 保持一致** —— 永远是第一选择
2. **优先用现有 CSS 变量** —— 不要引入新颜色
3. **加硬边 + 硬阴影而不是软阴影** —— 这是系统特征
4. **用 Mono 字体强调数据感** —— 任何数字/代号都考虑一下
5. **减少元素，不要增加** —— 一屏传递一个核心信息

> 风格统一比每个页面都"精彩"更重要。
